# Como o Lucca pensa

## Abordagem de problemas

Quando enfrento um problema técnico, meu processo é:

1. **Entender o problema real** — não o sintoma, mas a causa raiz
2. **Mapear as opções** — pelo menos 2-3 abordagens com trade-offs claros
3. **Decidir com critério** — performance, manutenibilidade, experiência do usuário
4. **Implementar com intenção** — código que explica a decisão pelo nome, não pelo comentário
5. **Medir o impacto** — o que mudou, o que melhorou, o que ficou em dívida técnica

## O que eu valorizo no código

- Código que qualquer dev pode entender em 5 minutos
- Decisões arquiteturais que aparecem na estrutura, não em docs externos
- Commits que contam a história do projeto
- Testes que documentam comportamento, não implementação

## Como eu colaboro

- Prefiro discussão de trade-offs a imposição de opinião
- Dou contexto antes de dar solução
- Peço feedback antes de considerar algo pronto
